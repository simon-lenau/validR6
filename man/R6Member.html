R6MemberR Documentation

Container for members in R6Class definitions.

Description

R6MemberRestriction.check A container that stores name, value and (optional) validity restrictions of type R6MemberRestriction for members to be used in R6Class declarations.

A R6Member object provides

  • a setter that checks restrictions

  • a getter returning the value

  • helper methods for using it as member (attribute or method) in R6Classes

See the sections on details, examples and methods.

Details

R6Member objects provide a formalized interface for setting up members of R6Classes and provide methods for automatically generating the corresponding R6Class() arguments public, private and active.

Validation rules for $value

Validation rules for the $value field are specified in the $restrictions field.

$restrictions contains a list of zero or more R6MemberRestrictions describing the validation rules for the $value field. Any new value is passed to the R6MemberRestriction$check() method of all elements in $restrictions before R6Member$value is changed. This ensures all validation criteria are met.

Combining multiple R6Members and declaring R6Classes

For declaring R6Classes, R6Members are most conveniently combined in a R6MemberList.

Examples

# ===================== > Create R6MemberRestrictions < ====================== #

# Restriction that checks if length of value equals 2
length_restriction <-
    R6MemberRestriction$new(
        id = "Length check",
        restriction = function(x) {
            if (length(x) != 2) {
                stop("Length should be 2!")
            }
            return(TRUE)
        }
    )

# Restriction that checks if type of value is "character"
type_restriction <-
    R6MemberRestriction$new(
        id = "Type check",
        restriction = function(x) {
            if (typeof(x) != "character") {
                stop("type should be character!")
            }
            return(TRUE)
        }
    )

# =========================== > Create R6Members < =========================== #

# R6Member object without restriction(s):
#   may take any value
R6Member_any <-
   R6Member$new(
       name = "example_member_any",
       value = 1:5
   )

# R6Member object with restriction(s):
#   all restriction(s) check(s) must return TRUE
R6Member_two_string <-
   R6Member$new(
       name = "example_member_char_len2",
       value = c("foo", "bar"),
       restrictions = list(
               "is length 2" = length_restriction,
               "is character" = type_restriction
       )
   )

# Initial value would violate restrictions
try({
R6Member_invalid <-
   R6Member$new(
       name = "example_member",
       value = 1:5,
       restrictions = list(
               "is length 2" = length_restriction,
               "is character" = type_restriction
       )
   )
})
#> Error in private$.err_violated_restrictions(value = value, restrictions = restrictions,  : 
#>   
Invalid attempt to set
#>    `example_member$value <-
#>        1:5`
#> Assignment does not meet the following restrictions:
#>    - Length check:
#>       Length should be 2!
#>    - Type check:
#>       type should be character!

Active bindings

is_method

Whether the R6Member should be treated as a method (function) member.
Typically determined automatically by checking if value is a function, but can be assigned as well. When TRUE, the member will be declared as a method in the R6Classes definition.
This field automatically resets every time value is set or changed, based on whether the new value is a function.

Type:   logical         
Default:  NULL
Affected methods: declare_active_binding
   declare_method
   declare_initialization

is_private

Determines whether the R6Member becomes a private member of the R6Classes.
If TRUE, the member is only accessible within the R6Classes object (e.g., in methods) and not exposed in the public interface. In this case, declare_active_binding and declare_method will return NULL. If FALSE (the default), the member is public and can be accessed externally.

Type:   logical         
Default:  FALSE        
Affected methods: declare_active_binding
   declare_method
   declare_initialization

locked

Determines whether the R6Member object's fields are locked (TRUE, the default) or writable (FALSE). When locked, most fields cannot be modified after initialization. This applies to all fields except itself, but can be overwritten for the value by setting value_locked Locking provides immutability guarantees and prevents accidental modifications to the member definition.

Type:   logical         
Default:  TRUE        
Affected methods: declare_active_binding
   declare_method
   declare_initialization

name

Name of the R6Member object. This name serves as the identifier for the member in the R6Classes definition.
Must be a valid R variable name and not be empty. The name is used to generate member declarations in public, private, and active binding contexts.

Type:   character         
Affected methods: declare_active_binding
   declare_method
   declare_initialization

private_name

The private internal name of the R6Member object. This is the name used to store the member in the private environment of the R6Classes. It is constructed from the public name with a prefix (default: .) and optional suffix.
This is a read-only field and is automatically computed from the public name.

Type:   character         

restrictions

A list of validity restrictions (R6MemberRestrictions) the R6Member's value must fulfill to be accepted. When the value is set or modified, it is checked against all restrictions in sequence. If any restriction fails, an error is raised and the value is not changed. An empty list means no restrictions are applied. See details and examples for more information on validation.

Type:   list         
Default:  NULL

value

Value of the R6Member object. Can be any R object (vector, list, function, etc.). If a function is assigned, the is_method field is automatically set to TRUE. It must meet all restrictions unless it is NULL. Any attempt to modify the value is first validated against all restrictions before the assignment is accepted.

Type:   any          
Default:  NULL

value_locked

Determines whether the value is locked (TRUE) or writable (FALSE).

Type:   logical         
Default:  FALSE        
Affected methods: declare_active_binding
   declare_method
   declare_initialization

Method accessor()

Usage
R6Member$accessor(field_value, field_name)
Arguments
field_value

Optional new value for field ⁠<field_name>⁠. If omitted the method returns the current field contents. If provided the field_value is validated and assigned to the field.

Type:   See active bindings for the valid type of each field.

field_name

The name of the internal field to be accessed.
See active bindings for a list of all fields in R6Member objects.

Type:   character         

Details

Access or modify one of the internal fields of R6Members

Returns

If field_value is provided (setter case):  NULL

Method print()

Usage
R6Member$print(value_level = TRUE, restriction_level = 0, print_class = TRUE)
Arguments
value_level

The amount of information about the value to be printed. One of:

value_level == FALSE or value_level <= 0 print nothing       
0 < value_level print value's first value_level characters
value_level == TRUE print value entirely
Type:   logical or integer       
Default:  TRUE        

When set to a positive number, this controls the maximum width for displaying the value. If `TRUE`, displays the full value. If `FALSE` or `0`, the value is not displayed.
restriction_level

The amount of information about each of the restrictions to be printed. One of:

  0 print nothing       
  1 print ids
  2 print ids and restriction functions
Type:   integer         
Default:  0         

Controls the detail level when printing restrictions. Higher values show more detailed restriction information. `0` means restrictions are not printed.
print_class

Whether the class name (R6Member) should be included when printing.

Type:   logical         
Default:  TRUE        

When TRUE, the class name is prepended to the member name in the output.

Details

Print an R6Member object to the console. Displays the member's name, optionally its value, and optionally its restrictions. Useful for inspecting member definitions and their validation rules.

Returns

NULL (invisibly). Output is printed to the console.
